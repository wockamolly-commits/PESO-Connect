UI / Interaction Pattern

How should the 7-step form look and feel?

A

Enhanced Current Style

Keep the existing card-based layout with the numbered step indicator at top. Upgrade with better field grouping, inline validation, and modern input components (searchable dropdowns, checkbox grids). Minimal disruption to existing design language.

Pros

  • Consistent with rest of app
  • Less work — reuse existing StepIndicator and styling
  • Familiar to returning users

Cons

  • Step indicator gets crowded at 7 steps on mobile
  • Card layout may feel dense with more fields
B

Modern Wizard with Progress Bar

Replace numbered circles with a slim progress bar + step title. Each step is a clean, spacious form section with grouped fields in subtle bordered sections. Floating "Back / Next" buttons pinned to bottom. More modern, breathing room.

Pros

  • Scales well to 7+ steps
  • Cleaner, less visual clutter
  • Better mobile experience
  • Progress bar feels more encouraging

Cons

  • Slightly different from current app style
  • Users can't see all step names at once
C

Hybrid — Progress Bar + Step Labels

Slim progress bar at top showing percentage, with compact step labels below (collapsible on mobile to show only current + adjacent steps). Best of both worlds — progress visibility with step context.

Pros

  • Users see where they are AND what's coming
  • Works well on both mobile and desktop
  • Progress bar is motivating
  • Step labels provide context

Cons

  • More complex step indicator component
  • Slightly more vertical space used
Recommendation: Option B (Modern Wizard with Progress Bar) — it scales best for 7 steps, feels clean and modern, and the slim progress bar takes less space than numbered circles while being more encouraging. The step title on each page provides enough context about where you are.
